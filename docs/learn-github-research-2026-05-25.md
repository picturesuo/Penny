# Learn GitHub Research Refresh - 2026-05-25

Scope: inspected 50 GitHub projects with recent activity across AI tutors, source-grounded study tools, NotebookLM-like apps, from-scratch AI curricula, prompt courses, and agent learning paths.

## Patterns To Use In Penny Learn

- Start with a diagnostic frame: identify the learner's current target, gap, and smallest useful next artifact before teaching.
- Use a from-scratch ladder for technical topics: primitive -> worked trace -> small test -> reusable artifact.
- Keep source-grounded learning visibly source-honest: source -> map -> teach -> use -> check, with claims marked when evidence is still needed.
- Add an operator lens without impersonation: translate public "how this kind of person thinks" into principles, constraints, inversion, and checks.
- Prefer Socratic teach-back and active recall over long generated explanations.
- Track weak spots locally: missed concepts, misconception, next drill, and mastery threshold.
- Let every lesson produce something durable: a code sketch, checklist, rule, prompt, flashcard, quiz, or Brain concept.
- Keep one compact screen per move; long curricula work because the ladder is visible and each step is small.
